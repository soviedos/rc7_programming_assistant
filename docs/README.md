## Convención de mantenimiento

Cualquier cambio en autenticación, arquitectura, interfaz, operación o testing debe reflejarse en esta documentación dentro de la misma iteración de desarrollo. La documentación describe el estado real del sistema, no aspiraciones futuras sin implementación.
